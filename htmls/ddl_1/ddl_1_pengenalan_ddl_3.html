Jenis - Jenis Perintah DDL

Perintah ini digunakan untuk membuat struktur baru dalam basis data, seperti tabel, indeks, atau tampilan.

  1. CREATE TABLE

    Membuat tabel baru dengan kolom dan tipe data yang ditentukan. Contoh sintaks: CREATE TABLE nama_tabel (kolom1 tipe_data, kolom2 tipe_data, ...);

  2. CREATE INDEX

    Membuat indeks pada kolom yang ditentukan untuk meningkatkan kecepatan pencarian data. Contoh sintaks: CREATE INDEX nama_indeks ON nama_tabel (nama_kolom);

  3. CREATE VIEW

    Membuat tampilan baru yang merupakan hasil dari suatu query. Contoh sintaks: CREATE VIEW nama_tampilan AS SELECT kolom1, kolom2 FROM nama_tabel;

Perintah ini digunakan untuk mengubah struktur basis data yang ada, seperti menambahkan kolom baru, menghapus kolom, atau mengubah tipe data kolom.

  1. ALTER TABLE

    Mengubah struktur tabel yang ada, seperti menambahkan kolom, menghapus kolom, atau mengganti nama kolom. Contoh sintaks: ALTER TABLE nama_tabel ADD kolom tipe_data;

  2. DROP

    Perintah ini digunakan untuk menghapus struktur basis data yang ada, seperti tabel, indeks, atau tampilan.

  3. DROP TABLE

    Menghapus tabel yang ada dalam basis data. Contoh sintaks: DROP TABLE nama_tabel;

  4. DROP INDEX

    Menghapus indeks yang ada pada kolom tertentu dalam tabel. Contoh sintaks: DROP INDEX nama_indeks ON nama_tabel;

  5. DROP VIEW

    Menghapus tampilan yang ada dalam basis data. Contoh sintaks: DROP VIEW nama_tampilan;

Perintah ini digunakan untuk menghapus semua baris dari tabel tanpa menghapus struktur tabel itu sendiri. Dibandingkan dengan perintah DELETE, TRUNCATE lebih cepat dan efisien karena tidak mencatat setiap penghapusan baris secara individual dalam log transaksi. Contoh sintaks: TRUNCATE TABLE nama_tabel;
